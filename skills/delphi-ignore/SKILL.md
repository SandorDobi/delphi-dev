# Skill: .claudeignore Manager for Delphi Projects

You are responsible for ensuring every Delphi project has a proper `.claudeignore`,
preventing binary, compiled, and IDE configuration files from being read unnecessarily.

## Language

Detect the language of the user's first message and respond **always in that language**.
Default: English.

## Automatic Execution Protocol

When detecting that the project contains `.dpr`, `.dproj` or `.pas` files:

### STEP 1 — Check existence
Check if `.claudeignore` exists in the project root.

### STEP 2A — If it does NOT exist
Create the `.claudeignore` file immediately with the default content below.
Then notify the user:

```
✅ .claudeignore created automatically.
Files ignored to save tokens:
- Binaries and compiled: .dcu, .exe, .dll, .bpl, .dcp, .rsm
- Resources: .res, .dres
- IDE configuration: .dproj, .dof, .cfg, .local
- Temporaries: .~*, .map, .drc
- Build outputs: Win32/, Win64/, Android/, iOSDevice64/, OSX64/
- IDE history: __history/
```

### STEP 2B — If it ALREADY exists but is incomplete
Compare with the default below. If relevant entries are missing, suggest an update:
"Your .claudeignore doesn't include [X]. Would you like me to update it?"

### STEP 3 — Do not alter source code files
The `.claudeignore` should never ignore: `.pas`, `.dfm`, `.dpr`, `.dpk`, `.inc`
These files contain source code and must always be read.

## What NOT to ignore (never add to .claudeignore)

| Extension | Reason |
|---|---|
| `.pas` | Pascal source code — primary file to read |
| `.dfm` | VCL form layout — important for understanding UI |
| `.dpr` | Project file — defines system units |
| `.dpk` | Package file — defines components |
| `.inc` | Includes — may contain relevant code |
| `.fmx` | FMX layout — important for FireMonkey projects |
